# 按自动化程度而非笔记类型拆分命令

`sync` 和 `audio` 命令的分界线是"是否需要人工介入"，而不是"处理哪种笔记"。菜谱同步和 AI链接同步都是全自动的，合并到 `sync` 命令；录音卡笔记需要用户在编辑器里审阅并选择目标数据库，因此单独为 `audio` 命令。这个划分让调度系统只需定时运行 `sync`，无需感知内部笔记类型的差异；`audio` 则由用户手动触发，保留了完整的交互控制权。

## Considered Options

按笔记类型拆分（`sync-recipe`、`sync-ai-link`、`audio`）也是备选方案，好处是每类可独立重跑，坏处是菜谱和 AI链接的自动化特征相同，强行拆分只增加运维负担而无实质收益。当前设计在此基础上保持了简洁性，代价是菜谱同步失败时 AI链接同步会被连带跳过，需依赖幂等重跑来恢复。
